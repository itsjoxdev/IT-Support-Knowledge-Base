# Password Reset Guide

## Overview

Password reset requests are one of the most common IT Help Desk tasks. This guide outlines the standard process for assisting users who have forgotten their password or need to reset it securely.

---

## Common Scenarios

- User forgot their password.
- Password has expired.
- User cannot sign in.
- Password does not meet complexity requirements.
- User wants to change their password.

---

## Possible Causes

- Forgotten password
- Expired password
- Incorrect username
- Keyboard Caps Lock enabled
- Account policy restrictions
- Synchronization issues

---

## Troubleshooting Methodology

### Step 1 – Verify User Identity

Before resetting any password:

- Verify the user's identity according to company policy.
- Confirm the username or email address.

---

### Step 2 – Check for Typing Issues

Verify:

- Caps Lock is off.
- Num Lock is correct.
- Correct keyboard language is selected.

---

### Step 3 – Confirm the Account Exists

Verify:

- Username is correct.
- The account has not been deleted or disabled.

---

### Step 4 – Reset the Password

Reset the password using the organization's approved identity management system.

After resetting:

- Require the user to change the password at the next sign-in if applicable.

---

### Step 5 – Verify Login

Ask the user to:

- Sign in using the temporary or new password.
- Change the password if required.

---

## Password Best Practices

A strong password should:

- Be at least 12 characters long.
- Include uppercase and lowercase letters.
- Include numbers.
- Include special characters.
- Avoid personal information.
- Be unique for every account.

---

## Resolution

Password reset is considered successful when:

- The user can sign in successfully.
- The password has been updated.
- The user confirms access to required systems.

---

## Prevention

- Use a password manager.
- Enable Multi-Factor Authentication (MFA).
- Do not reuse passwords.
- Change passwords only when required by company policy.
- Never share passwords.

---

## Escalation

Escalate the issue if:

- Identity cannot be verified.
- The account is disabled or deleted.
- Password reset fails.
- Multiple failed login attempts indicate a possible security incident.
