# Password Policy

## Overview

A strong password policy helps protect user accounts and organizational data from unauthorized access. This guide outlines recommended password requirements and security best practices commonly followed in IT environments.

---

## Password Requirements

A secure password should:

- Be at least 12 characters long.
- Contain uppercase letters (A–Z).
- Contain lowercase letters (a–z).
- Include numbers (0–9).
- Include special characters (e.g., ! @ # $ % ^ & *).
- Avoid dictionary words and common phrases.
- Be unique for each account.

---

## Password Rules

Users should:

- Never share passwords.
- Never write passwords on paper.
- Never reuse passwords across different accounts.
- Never store passwords in plain text.
- Change passwords immediately if compromise is suspected.

---

## Multi-Factor Authentication (MFA)

Whenever possible:

- Enable Multi-Factor Authentication (MFA).
- Use an authenticator app instead of SMS when supported.
- Keep backup authentication methods secure.

---

## Password Storage

Recommended practices:

- Use a trusted password manager.
- Do not save passwords in web browsers on shared computers.
- Protect password manager accounts with a strong master password.

---

## Common Weak Password Examples

Avoid passwords such as:

- password123
- 12345678
- qwerty
- admin
- welcome123
- your name or birth date

---

## Strong Password Example

Example:

```
J0x!Secure2026#
```

**Note:** This is only an example. Every password should be unique.

---

## Password Reset Best Practices

Before resetting a password:

- Verify the user's identity.
- Follow the organization's security policy.
- Require a password change after issuing a temporary password, if applicable.

---

## Security Recommendations

- Enable account lockout after multiple failed login attempts.
- Review password policies regularly.
- Monitor for suspicious login activity.
- Educate users about phishing attacks.
- Encourage the use of passphrases for improved security.

---

## Resolution

A secure password policy reduces the risk of:

- Unauthorized access
- Account compromise
- Credential theft
- Brute-force attacks
- Password reuse attacks
